JavaScript Fundamentals

The relationship between HTML, CSS and Javascript

Website are mostly built from three languages - HTML, CSS and Javascript. In fact, these languages are often referred to as the 'building blocks of the web'. I'm going to break down what each of these languages do by thinking about a pop-up book.

HTML - What's in it?

HTML stands for Hyper Text Markup Language and it is the language we use to get content and the structure of that content, onto the page of a website. We can use HTML to instruct our browser to display things like a page heading, a sub-heading, paragraphs, pictures and lists. We use HTML to control the order those things appear in, and the actual words and pictures that are included.

In our pop-up book analogy, HTML would be responsible for the fact that the book has a title on the cover, maybe a dedication on the next page, and then some paragraphs on each page that follows.

CSS - How does it look?

CSS stands for Cascading Style Sheets and is the language we use to instruct our browser how the page and the content of our website should look. We can control where on the page things are positioned, how much space each thing has around it what colour they are, what the colour of the background is, what fonts are used and so on.

Without the CSS, our book would look pretty plain - in fact, other than the original content, it would look almost exactly the same as every other book that has only used HTML so far. The CSS means that our book has colour, patterns and different fonts - in short, it looks beautiful and pretty unique!

JavaScript - What does it do?

JavaScript is a programming language that can access the things put in place on a page with HTML and CSS, and make them change under certain conditions. JavaScript is what makes things on a website respond to certain events and to allow users of the website to interact with it.

For our pop-up book, JavaScript is what distinguishes our book from being just an ordinary picture book. Turning the page is an event or an interaction which causes a portion of the page to pop-up when that event happens. If the page isn't turned, that portion of the page doesn't move or change.

Control flows and loops

JavaScript code typically works from top to bottom when it is executing. Control flow is used in JavaScript to change this behaviour and determine which part of the code should run, and when.

One tool in control flow is the use of conditional statements. Conditional statements allow us to only have a piece of code run if a certain condition is met.

For example:

If the button is clicked once, run the code that displays a message saying "Click!"
If the button is not clicked, do not run the code.

Loops are a type of conditional statement that enable repetition in the code when certain conditions are met, without the code having to be written out multiple times. With a loops, we can move through collections of data and make different actions happen with that data when we want them to.

Humans use control flow because we adapt to the conditions around us. Let's think about going skiing.

If there was no control flow involved:

1. Ride chairlift to the top
2. Ski to the bottom.

However, this isn't practical because certain conditions might make the experience better or worse and we might want to do a second run, depending on how we feel or what the time is.

With control flow involved, skiing actually looks more like this:

1. Check if the conditions on the mountain are good. If they are, travel to the ski-field. If they aren't, think of something else to do today.
2. Is the chair lift working? If it is, get on it. If it isn't, ask for a refund and go home or wait around to see if it gets fixed.
3. Are your skills best matched to run A or run B? Then do the run that matches your skill level.
4. Once you are at the bottom, decide whether you have a) enough time and b) enough energy to do another run. As long as you have enough, keep doing runs. When you don't have enough of either of them, stop doing runs.

In short, control flow means that processes become dynamic and adaptable as opposed to static. When used in Javascript, we end up with websites that are responsive, interesting and varied.

What is the DOM?

The Document Object Model, frequently shortened to the DOM, is a programming interface that organises a web page into a structure of things called nodes. Nodes can have parents and children.

Arrays and Objects

To understand arrays and objects in JavaScript, we first need to understand variables. Variables are containers for storing data values and are typically created in this way:

let names = ["Jennifer", "Cameron", "Sam", "Thomas", "Lilly"]

Most importantly, each item in that array is given an index position, starting from 0. If I wanted to access Thomas' name later on in my script, I would need to refer to it as:

names[3]

Objects are constructed slightly differently, in that under one variable name, other variables are listed in value pairs. If I wanted to store details about myself in an object, it would look like this:

let myDetails = {firstName: "Chelsea", lastName: "Liddy", age: 30, location: "Christchurch"}

The data is not all of the same type and there are value pairs inside the object.

To access my age, I first specify that I want the myDetails object, then that I want the age value. This would look like:

myDetails.age

Functions

Functions in JavaScript are sets of statements that are created under one name. The statements make up the instructions for how to perform a certain task.
